# Project Title
Customer Service (CS) Capacity Plannerr (FMProj_CSCapacityPlanner)


## Features
- Interactive workforce capacity planning calculator.
- Flexible planning assumptions and configurable inputs.
- Real-time staffing estimates and scenario modelling.
- Visual summaries of simulation and stress-testing results.
- Browser-based application with no installation required.
- Anonymous by design — no sign-in and no data stored.

## What It Does
- Estimates workforce requirements for projected customer service workloads.
- Converts operational data into AI-assisted staffing recommendations.
- Compares available capacity against expected demand.
- Enables rapid evaluation of multiple planning scenarios.

## Intended Purposes
- Simplifies workforce planning and operational forecasting.
- Supports staffing decisions using statistical models including Poisson, Erlang C, and Monte Carlo simulation.
- Reduces reliance on manual calculations and spreadsheets.
- Identifies potential capacity gaps before they affect service delivery.
- Promotes data-driven planning through a private, browser-based experience.

## Reflections (Why I Created This App)
Throughout my career in customer service operations, one recurring challenge has been determining the right staffing levels—balancing operational costs against consistently achieving service KPIs. Producing reliable capacity estimates often requires significant time and analysis, especially when supporting management decisions.
This application was created to streamline that process. While it does not address every real-world scenario, it provides rapid, evidence-based estimates that reduce initial guesswork and serve as a practical starting point for further planning and decision-making.

## Technical Challenges
- Frustration level: 7/10.
- The project was split roughly 50% frontend development and 50% AI/backend integration.
- Integrating AI to analyse simulation outcomes against service KPIs proved more complex than expected.
- Debugging frontend-backend communication, including API requests and responses, required considerable effort.
- Designing a responsive interface capable of presenting complex analysis clearly was an unexpected challenge.
- Balancing AI capability against token usage and cost required careful optimisation, as more capable models generally deliver better results but come with higher costs and stricter usage limits.

## Current Status
Approximately 98% of known bugs have been resolved. The application is now largely stable, but it has yet to undergo comprehensive stress testing to validate its reliability, consistency, and performance under heavier workloads.

### Disclaimer: This application is provided as-is for demonstration and educational purposes. While every effort has been made to ensure accuracy, results should be independently verified before being used for operational or business decisions.
